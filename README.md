# Прототип системы детекции уборки столиков по видео

Прототип для отслеживания состояний столика (Свободен/Занят) и подсчета среднего времени между уходом одного гостя и подходом следующего. Решение построено на архитектуре конечного автомата (State Machine) для обеспечения стабильности метрик и устранения ложных срабатываний из-за кратковременных перекрытий.

## Используемый стек

- Python 3.8+
- OpenCV (работа с видео, ручное выделение ROI, визуализация)
- Ultralytics YOLOv8 (детекция людей)
- Pandas (сбор статистики и отчетов)

## Особенности архитектуры

Логика разделена на 4 компонента в соответствии с SOLID (single responsibility):
1. `ObjectDetector`: отвечает исключительно за вызов YOLOv8 и извлечение box-координат объектов класса "человек" (0).
2. `TableTracker`: конечный автомат с механизмом фильтрации мерцаний (debouncing). Не позволяет состоянию стола постоянно прыгать при кратковременном перекрытии человека. Статус меняется только при устойчивом присутствии/отсутствии в зоне (по умолчанию от 2 секунд).
3. `AnalyticsEngine`: логирует события переходов в Pandas DataFrame и рассчитывает среднее время задержки.
4. `VideoProcessor`: оркестратор ввода-вывода (OpenCV).

## Как запустить

1. Создайте и активируйте виртуальное окружение:
   ```bash
   python -m venv venv
   # В Windows: 
   venv\Scripts\activate
   # В Linux/macOS:
   # source venv/bin/activate
   ```

2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

3. Запустите скрипт, передав путь к видео (например, `video.mp4`):
   ```bash
   python main.py --video video.mp4 --output output.mp4
   ```

4. При запуске откроется окно первого кадра видео (`Select Table`). 
   - Выделите один целевой столик с помощью левой кнопки мыши.
   - Нажмите `ENTER` или пробел для подтверждения. 
   - Скрипт начнет обработку с прогресс-баром в консоли.

## Пример отчета

**Логика:**
Используется модель YOLOv8n (nano). При пересечении bounding box человека и области интереса столика запускается внутренний таймер задержки. Если человек находится в зоне больше 2-х секунд, стол помечается как `OCCUPIED`. Если покидает зону — `EMPTY`. 

В финале выводится подробный отчет о всех эпизодах смены статуса с расчетом средних величин. Итоговый файл `output.mp4` содержит зеленую рамку для пустого стола и красную — для занятого.

*(Здесь можно добавить скриншот "проблемного кадра", где происходит сбой детекции из-за перекрытий, если таковой был замечен в ходе ручного прогона).*
